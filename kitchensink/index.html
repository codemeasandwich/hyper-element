hyper-element Kitchen Sink

Interactive examples demonstrating every feature of hyper-element. Each page is also an automated E2E test case.

Core Features

  • Basic Rendering Simple element rendering with Html template
  • Async Setup Using setup() for async data initialization
  • Lifecycle Sequence Order of setup(), render(), teardown() calls

Attributes & Data

  • Attributes Accessing element attributes via this.attrs
  • Observed Attributes Using static observedAttributes for reactive updates
  • Dataset Using data-* attributes with JSON parsing
  • Dataset Mutations Modifying dataset and re-rendering
  • Type Coercion Automatic parsing of numbers, booleans, JSON

Content Handling

  • Wrapped Content Accessing content between element tags
  • Content Mutations Re-rendering when content changes

Templates & Fragments

  • Templates Using Html.template() with variables
  • Advanced Templates Using {+if}, {+each}, {+unless} block syntax
  • Block Syntax Advanced Nested blocks, edge cases, null/undefined handling
  • Auto-Wire Each Using {+each ${'{array}'}}...{-each} in templates
  • Async Fragments Fragments with promises and placeholders
  • Fragment Templates Fragments with template strings and values
  • Fragment Combinations All fragment result types: text, html, any, template

Rendering

  • Keyed Rendering Html.wire() for efficient keyed list updates
  • XSS Prevention HTML escaping and Html.raw() for safe rendering

Composition

  • Nested Elements Parent-child custom element composition
  • Child Redraw Parent-child attribute passing and re-renders
  • Child Styles Passing style objects to children
  • Complex Type Attributes Passing objects and functions to child elements

Events

  • Event Callbacks Using setup() for external event handling

Edge Cases

  • Corner Cases Edge cases and unusual scenarios
  • Coverage Edge Cases Additional edge cases for code coverage

Performance

  • Stress Test Large lists, rapid updates, deep nesting
